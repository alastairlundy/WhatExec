## Prerequisites
Before you begin, ensure you have the following installed:
- [.NET SDK](https://dotnet.microsoft.com/download) (latest stable version recommended)

## How to Contribute

### Reporting Bugs
If you find a bug, please [open a GitHub issue](https://github.com/alastairlundy/whatexec/issues/new) and provide as much detail as possible, including steps to reproduce the issue.

### Suggesting Features
Discussion of adding new features via a GitHub issue prior to commencing work on not explicitly planned features is not required but greatly appreciated. This helps ensure that the proposed changes align with the project's goals and avoid duplicated effort.

### Pull Requests
When you're ready to submit your changes:
1. Fork the repository.
2. Create a new branch for your feature or bug fix.
3. Commit your changes.
4. Push to your fork and submit a Pull Request.

## Guidelines

### Pull Request Scope
Pull Requests (PRs) should be small and focused on a single issue (e.g., "Fixes a bug" or "Adds a new feature"). This makes the review process easier and helps maintain code quality. Avoid submitting large PRs that address multiple unrelated issues.

### AI-Generated Code Policy
To maintain high-quality contributions and clear authorship:
- Contributions should not be made of entirely AI-generated code.
- Pull Requests that include AI-generated code must explicitly declare which portions of the code were generated by AI.

### Licensing
By contributing to this project, you agree that your contributions (where code is written by a human) will be licensed under the [Mozilla Public License 2.0 (MPL 2.0)](LICENSE.txt).

## Development Setup
1. Clone the repository:
   ```bash
   git clone https://github.com/alastairlundy/whatexec.git
   cd whatexec
   ```
2. Build the project:
   ```bash
   dotnet build
   ```
3. Run tests (if applicable):
   ```bash
   dotnet test
   ```
